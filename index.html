    • HIV World Prevalence Overview

    In order to raise the public awareness of the HIV/AIDS prevention and find out effective approaches to alleviate its spreading, our team decides to implement an N-tier web application to visualize related data. By mapping groups of data to a graph, our application should be able to tell a story about recent trends of HIV/AIDS and potential correlated factors. By digging into open data, such as The World Bank data and The Open US data, our application is able to promote government transparency and facilitate the policy makers. Our team plans to design and build a N-tier web application to visualize the HIV prevalence and its correlated factors. The application is an interactive design based on javascript libraries. Users are able to interact with a Choropleth Map page, a Line Chart page, a Bubble Chart page and a customized preference page embedded in this system. By interacting with this application, users can investigate the current trends and prevalence of the global HIV infection.

    HIV Prevalence Overview

    I Introduction

    This is a N-tier web application based on Ruby on Rails that serves to visualize the HIV prevalence and its coorelated factors. The application is an interactive design based on Google Visualization APIs and javascript libraries. Users are able to interact with a Cholopleth Map page, a Line Chart page, a Bubble Chart page and a customized perferrence page embedded in this system. Google News, Google Youtube and Wikipedia APIs are also integrated. Users by interacting with this application will be informed of the current situations and trends of the global Hiv infection as well as impacts of related indicators.

    II System Requirements

    • Supported Operating Systems: GNU/Linux, Microsoft Windows 7 64- bit, Macintosh OS 64-bit
    • Recommended Server: Rails v3.2.0
    • Recommended Database: MongoDB v2.2.0
    • Recommended Browser: Google Chrome v23.0, Firefox v16.0.2 (Please make sure Javascript is enabled) NOTE: IE may not be supported currently

    III Deployment for Testing Puropose

    • Please ensure that Ruby, Rails, RubyGem, MongoDB is successfully installed and configured.
    • Unzip the project files to a local folder
    • Setup database: In a terminal, run "mongod"
    • Open a terminal and cd into a project folder, run "bundle" (bson_ext may fail in Windows cmd, please use MinGW instead)
    • Initialize database: run "rake db:seed" (if failed, use "bundle update" instad in last step.)
    • Open a new terminal, cd into project folder and run "rails server"
    • Test: Open a new browser, input url "http://127.0.0.1:3000/"
